## Нефункциональные требования

| ID | Требование |

| NFR-1 | Исходный код проекта должен храниться в GitHub-репозитории |

| NFR-2 | Разработка должна вестись с использованием отдельных веток |

| NFR-3 | Изменения должны попадать в основную ветку разработки через Pull Request |

| NFR-4 | Клиентская и серверная части должны быть разделены |

| NFR-5 | Обмен данными между frontend и backend должен выполняться в формате JSON |

| NFR-6 | Проект должен запускаться локально на компьютере разработчика |

| NFR-7 | Backend должен предоставлять REST API |

| NFR-8 | Данные о пользователях, товарах, корзине и заказах должны храниться в базе данных |

| NFR-9 | Пароли пользователей не должны храниться в открытом виде |

| NFR-10 | Проект должен иметь понятную инструкцию по запуску |

## Трекер задач

Для управления задачами проекта используется GitHub Issues.

Каждая задача оформляется как отдельный Issue. Для задач используется следующая структура названий:

- `DOCS` — документация проекта

- `PROJECT` — настройка структуры проекта

- `AUTH` — авторизация и регистрация

- `CATALOG` — каталог товаров

- `CART` — корзина

- `ORDER` — оформление заказов

- `TEST` — тестирование

- `DEVOPS` — сборка, Docker и запуск проекта

## Планирование спринтов

Разработка проекта делится на несколько спринтов. 

### Спринт 1. Подготовка проекта

Цель спринта: подготовить основу проекта и организовать процесс разработки.

Задачи спринта:

- сформировать команду и распределить роли;
- описать функциональные и нефункциональные требования;
- выбрать трекер задач;
- создать GitHub Issues;
- настроить ветки `main`, `develop`, `auth`, `catalog`, `cart`;
- описать правила работы с Git;
- подготовить базовую структуру проекта.

### Спринт 2. Архитектура и каркас приложения

Цель спринта: определить архитектуру проекта и подготовить каркас клиентской и серверной частей.

Задачи спринта:

- описать архитектуру проекта;
- описать API-контракты взаимодействия frontend и backend;
- создать каркас frontend-приложения;
- создать каркас backend-приложения;
- настроить запуск backend;
- подготовить структуру папок;
- настроить подключение к базе данных.

### Спринт 3. Базовый функционал

Цель спринта: реализовать основные пользовательские сценарии интернет-магазина.

Задачи спринта:

- реализовать регистрацию пользователя;
- реализовать авторизацию пользователя;
- реализовать просмотр каталога товаров;
- реализовать просмотр карточки товара;
- реализовать добавление товара в корзину;
- реализовать просмотр корзины.

### Спринт 4. Завершение проекта и тестирование

Цель спринта: завершить разработку, протестировать приложение и подготовить проект к защите.

Задачи спринта:

- реализовать изменение количества товара в корзине;
- реализовать удаление товара из корзины;
- реализовать оформление заказа;
- выполнить тестирование основных сценариев;
- исправить найденные ошибки;
- подготовить инструкцию по запуску;
- подготовить итоговый отчет.
