## Архитектура проекта

Проект реализуется по клиент-серверной архитектуре.

Клиентская часть отвечает за отображение пользовательского интерфейса и взаимодействие пользователя с системой. Серверная часть обрабатывает запросы клиента, выполняет бизнес-логику и работает с базой данных. База данных используется для хранения информации о пользователях, товарах, корзине и заказах.

### Общая схема архитектуры

```text
Пользователь
    |
    v
Frontend-приложение React
    |
    | HTTP-запросы / JSON
    v
Backend-приложение Node.js + Express
    |
    | Prisma ORM
    v
База данных PostgreSQL
```

### Компоненты системы

| Компонент | Технология | Назначение |
|---|---|---|
| Frontend | React + TypeScript | Пользовательский интерфейс интернет-магазина |
| Backend | Node.js + Express + TypeScript | REST API, бизнес-логика, обработка запросов |
| Database | PostgreSQL | Хранение данных пользователей, товаров, корзины и заказов |
| ORM | Prisma | Работа backend-приложения с базой данных |
| Containerization | Docker Compose | Локальный запуск сервисов проекта |

### Frontend

Frontend отвечает за пользовательский интерфейс приложения.

Основные страницы frontend-приложения:

- страница регистрации;
- страница авторизации;
- страница каталога автозапчастей;
- страница карточки товара;
- страница корзины;
- страница оформления заказа.

Frontend отправляет HTTP-запросы на backend и получает ответы в формате JSON.

### Backend

Backend предоставляет REST API для работы приложения.

Основные зоны ответственности backend:

- регистрация и авторизация пользователей;
- получение списка товаров;
- получение информации о конкретном товаре;
- работа с корзиной пользователя;
- оформление заказа;
- взаимодействие с базой данных.

### Database

В базе данных хранятся основные сущности проекта:

- пользователи;
- товары;
- элементы корзины;
- заказы;
- товары в заказах.

## Правила работы с Git

В проекте используется модель ветвления, похожая на Git Flow.

Основная разработка ведётся не напрямую в ветке `main`, а через отдельные ветки.

### Основные ветки проекта

| Ветка | Назначение |
|---|---|
| `main` | Стабильная версия проекта |
| `develop` | Основная ветка разработки, в которую объединяются готовые изменения |
| `auth` | Ветка для разработки регистрации и авторизации |
| `catalog` | Ветка для разработки каталога товаров |
| `cart` | Ветка для разработки корзины и оформления заказа |

### Правила работы с ветками

1. Ветка `main` используется только для стабильной версии проекта.
2. Все новые изменения сначала попадают в ветку `develop`.
3. Для разработки отдельных модулей используются feature-ветки: `auth`, `catalog`, `cart`.
4. Изменения из feature-веток объединяются в `develop` через Pull Request.
5. Перед началом работы необходимо обновить свою ветку актуальными изменениями из `develop`.
6. Перед созданием Pull Request необходимо проверить, что проект запускается без ошибок.

### Порядок работы разработчика

1. Переключиться на свою ветку.
2. Получить актуальные изменения из `develop`.
3. Выполнить задачу.
4. Создать коммит с понятным сообщением.
5. Отправить изменения в удалённый репозиторий.
6. Создать Pull Request в ветку `develop`.

### Правила Pull Request

Каждый Pull Request должен содержать краткое описание выполненной работы.

В Pull Request необходимо указать:

- какая задача была выполнена;
- какие файлы были изменены;
- как проверить результат;
- есть ли ограничения или недоделанные части.

## Авторизация

### POST `/auth/register`

Регистрация нового пользователя.

Тело запроса:

```json
{
  "name": "Иван Иванов",
  "email": "ivan@example.com",
  "password": "123456"
}
```

Успешный ответ:

```json
{
  "id": 1,
  "name": "Иван Иванов",
  "email": "ivan@example.com",
  "token": "jwt-token"
}
```

Возможные ошибки:

```json
{
  "message": "Пользователь с таким email уже существует"
}
```

---

## Корзина

Для работы с корзиной пользователь должен быть авторизован.

Во все запросы корзины передается заголовок:

```text
Authorization: Bearer jwt-token
```

### GET `/cart`

Получение содержимого корзины пользователя.

Успешный ответ:

```json
{
  "items": [
    {
      "id": 1,
      "productId": 1,

      "articleNumber": "OF-2107",
      "price": 750,
      "quantity": 2,
      "total": 1500
    }
  ],
  "totalPrice": 1500
}
```

```json
}
```

