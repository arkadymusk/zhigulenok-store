Онлайн-магазин автозапчастей для автомобилей Лада

## Функциональные требования

| ID | Требование |

| FR-1 | Пользователь может зарегистрироваться в системе |

| FR-2 | Пользователь может авторизоваться в системе |

| FR-3 | Пользователь может выйти из аккаунта |

| FR-4 | Пользователь может просматривать каталог автозапчастей |

| FR-5 | Пользователь может искать товары по названию или артикулу |

| FR-6 | Пользователь может фильтровать товары по категории |

| FR-7 | Пользователь может открыть карточку товара |

| FR-8 | Пользователь может добавить товар в корзину |

| FR-9 | Пользователь может просматривать содержимое корзины |

| FR-10 | Пользователь может изменять количество товара в корзине |

| FR-11 | Пользователь может удалить товар из корзины |

| FR-12 | Пользователь может оформить заказ |

## Нефункциональные требования

| ID | Требование |

| NFR-1 | Исходный код проекта должен храниться в GitHub-репозитории |

| NFR-2 | Разработка должна вестись с использованием отдельных веток |

| NFR-3 | Изменения должны попадать в основную ветку разработки через Pull Request |

| NFR-4 | Клиентская и серверная части должны быть разделены |

| NFR-5 | Обмен данными между frontend и backend должен выполняться в формате JSON |

| NFR-6 | Проект должен запускаться локально на компьютере разработчика |

| NFR-7 | Backend должен предоставлять REST API |

| NFR-8 | Данные о пользователях, товарах, корзине и заказах должны храниться в базе данных |

| NFR-9 | Пароли пользователей не должны храниться в открытом виде |

| NFR-10 | Проект должен иметь понятную инструкцию по запуску |
